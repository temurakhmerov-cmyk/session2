# Управление очередью печати на основе Приоритетной Очереди (Двоичной Кучи)

Этот проект представляет собой реализацию системы управления очередью печати с использованием **собственной двоичной кучи (Min-Heap / Binary Heap)** для приоритетной очереди. Проект разработан как учебное задание по дисциплине «Структуры данных и алгоритмы» на языке Python.

---

## 📖 Теоретическая часть

### Приоритетная очередь (Priority Queue)
Приоритетная очередь — это абстрактный тип данных, который поддерживает операции добавления элементов и извлечения элемента с наивысшим приоритетом (в данном проекте — с наименьшим числовым значением приоритета: 1 — Высокий, 2 — Средний, 3 — Низкий).

### Двоичная куча (Binary Heap)
Двоичная куча — это древовидная структура данных, представленная в виде почти полного бинарного дерева, которая удовлетворяет **свойству кучи**:
* **Min-Heap**: приоритет родителя всегда меньше либо равен приоритету его дочерних элементов (значение в корне дерева является минимальным в куче).

#### Представление кучи в памяти
Поскольку двоичная куча является почти полным деревом, ее можно эффективно хранить в виде плоского массива (списка) без использования указателей на узлы. Для элемента с индексом `i`:
* Индекс родителя: `parent(i) = (i - 1) // 2`
* Индекс левого сына: `left(i) = 2 * i + 1`
* Индекс правого сына: `right(i) = 2 * i + 2`

#### Основные алгоритмические операции
1. **Вставка (`insert`)**: элемент добавляется в конец массива, после чего запускается процедура просеивания вверх (`sift_up`) до тех пор, пока не восстановится свойство кучи.
2. **Извлечение минимума (`extract_min`)**: значение корня (индекс 0) заменяется последним элементом массива. Последний элемент удаляется, а новый корень просеивается вниз (`sift_down`) для восстановления свойств кучи.
3. **Стабильность кучи (FIFO)**: в классической куче стабильность не гарантируется. Чтобы задачи с одинаковым приоритетом обрабатывались в порядке их добавления (First-In-First-Out), каждый элемент кучи хранится в виде кортежа: `(priority, order_id, job)`, где `order_id` — последовательно возрастающий счетчик (уникальный для каждой операции вставки). Сравнение кортежей в Python происходит поэлементно: при равных `priority` сравниваются `order_id`, что гарантирует FIFO.

### Временная сложность операций
| Операция | Сложность в худшем случае | Описание |
|---|---|---|
| **Вставка (`insert`)** | $O(\log N)$ | Просеивание вверх по высоте дерева |
| **Извлечение минимума (`extract_min`)** | $O(\log N)$ | Просеивание вниз от корня до листьев |
| **Просмотр вершины (`peek`)** | $O(1)$ | Элемент всегда находится по индексу `[0]` |
| **Очистка кучи (`clear`)** | $O(1)$ | Очистка массива |

## 🛠️ Структура проекта

* `queue_sys/heap.py` — собственная реализация двоичной кучи (Min-Heap) со счетчиком порядка для стабильности очереди.
* `queue_sys/models.py` — классы объектов печати (`PrintJob`) и принтеров (`Printer`) с методами симуляции времени.
* `queue_sys/manager.py` — класс `PrintQueueManager` координирующий очередь и свободные принтеры.
* `app.py` — веб-сервер на FastAPI для интерактивной визуализации и API.
* `cli.py` — консольная текстовая версия симуляции для демонстрации в терминале.
* `static/` — файлы веб-интерфейса (HTML, CSS, JS):
  * `index.html` — разметка интерактивной панели.
  * `style.css` — стильное оформление в темных тонах (стеклянный интерфейс, градиенты, анимация).
  * `app.js` — отрисовка двоичной кучи в виде дерева (SVG) и массива, отправка API запросов, периодическое обновление.

## 🚀 Как запустить проект

### Требования
* Python 3.8 или выше.

### Шаг 1. Установка зависимостей
Установите необходимые библиотеки для работы веб-сервера:
```bash
pip install -r requirements.txt
```

### Шаг 2. Запуск консольной версии (CLI)
Для быстрой демонстрации работы структуры данных без запуска веб-интерфейса выполните:
```bash
python cli.py
```
Консоль покажет пошаговый процесс добавления 5 задач разного приоритета и их распределения по 2 принтерам с шагом времени.

### Шаг 3. Запуск интерактивного веб-визуализатора
Запустите веб-сервер FastAPI:
```bash
python -m uvicorn app:app --reload
```
Или просто:
```bash
uvicorn app:app --reload
```

После этого откройте браузер по адресу:
👉 **[http://127.0.5.1:8000](http://127.0.0.1:8000)** (или `http://localhost:8000`)

В браузере откроется панель управления, где вы сможете:
1. Добавлять свои задачи с указанием автора, объема (страниц) и приоритета.
2. Отслеживать состояние кучи в виде интерактивного **двоичного дерева (SVG)** или плоского **массива памяти**.
3. Наблюдать за ходом печати принтеров в реальном времени (прогресс-бар).
4. Отключать/включать принтеры кнопкой 🚫/🔌 (если принтер отключается, его активная задача автоматически возвращается обратно в кучу).
5. Использовать быстрые кнопки генерации случайных задач или сброса системы.
